# DropPad Roadmap

This document outlines the planned development and future vision for DropPad.

## 🟢 Near-Term Improvements (Next Release)
- **WebSockets / Durable Objects**: Instant real-time updates for notes and file uploads.
- **Improved E2E Testing**: Full Playwright coverage for all critical user flows.
- **Custom Expiry Times**: Allow users to select 10m, 1h, or 24h expiration.
- **Mobile PWA**: Optimize the mobile experience and support installable app manifests.

## 🟡 Advanced Future Ideas
- **Client-Side Encryption**: Encrypt all notes and files in the browser before they reach the edge.
- **QR Code Scanning**: Built-in QR scanner to quickly join a workspace from a mobile device.
- **CLI Tool**: A lightweight CLI (`dp upload <file>`) for instant terminal sharing.
- **VSCode Extension**: Drag-and-drop snippets directly from the editor into a DropPad workspace.

## 🔴 Non-Goals
- **Permanent Storage**: DropPad is and will always be temporary.
- **User Accounts / Auth**: Keeping the barrier to entry at zero is a core principle.
- **Social Features**: No commenting, liking, or social feeds.
- **Giant File Hosting**: Not a replacement for dedicated large-file services.
